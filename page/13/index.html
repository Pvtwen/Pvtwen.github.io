随笔

发表于 2021-03-27 | 更新于: 2023-02-25

记录一些操作手册性质的东西

  1. 如何条件调试断点 例如停在循环的某一步 如何回到上一步? 参考链接
  • 调试中的drop frame作用：丢掉当前函数的栈帧，也就是回到上一步的调用，但是并不能回到本次循环的调用
  1. 电脑作主机，让手机连接梯子操作：一 电脑热点 二 打开手动代理 ip设置为127.0.0.1 端口号设置为7890 三 找到电脑热点主机ip 四 手机连接热点 手动代理，主机号和端口号分别设置为以上主机的ip和7890
  • 1.17日git操作回忆,gitee为例，.git文件中保存着本地仓库与remote仓库的连接，所以这个文件是关键，如果想要把自建工程上传到远程仓库，.git文件一定要处理好，可以用以下办法处理，git clone httpsx到本地,.git文件必须复制,其余的选择性复制.最后git commit -m “”,git push origin master.
  • github中git clone https无法使用，可能是本地的权限更改，google解决不了。但是可以使用ssh git clone.
  • 无论是gitee还是github上传项目，都应该遵从第一点的办法，用官网提供的方法会报错。
  • git pull的操作只会把远程分支最近的更新pull到本地，而不是像clone那样将全部项目的内容clone到本地。
  • 更改social link图标，next/config.yml中social links，查阅font awesome手册添加
  • 2.7日 重装系统之后的博客更新，步骤如下
  • 最终解决办法：npm install hexo-cli -g, hexo init blog, cd blog, npm install, hexo s,git clone之前的备份，node_module,.git文件可以不要，其他的原样复制.因为hexo deploy并不会完全覆盖之前的仓库，css会混淆，所以删除gitee对应的仓库，重建，若仓库名有变化，在hexo-config.yml中更改repo和url.若报错，查阅上述debug
  • 若首页中的日志按钮无法使用，在 Next 主题中，归档页面的默认 URL 地址为 /archives/，而不是 /archives||archive。更改即可。
  • 若直接使用备份的blog，可能出现以下报错，hexo g，hexo s之后ERROR Database load failed. Deleting database.,这时，回到上述最终解决办法中，重新hexo init blog新建立一个博客，之后将备份文件中的除了node_modules,.github,.delopy_git文件夹的其他文件夹拷贝，但是themes主题下的next主题最好只拷贝config.xml文件，因为有可能之前的next文件夹缺少相关依赖.
  • 更改next的config文件中的updatedtime设置文章更新日期.
  • node:19.0.0 hexo:5.4.0(更改hexo版本号 npm i hexo@5.4.0) npm:8.19.2
